The skills

Seven skills, cleanly split — two reviews (architecture vs diff), two audits (app vs tests).

dte-arc-auditor

Audit a whole app → a target architecture + a phased, validated, gated migration roadmap + worklist.

dte-arc-review

Review a slice or structure. Severity-ranked architectural findings (diagnostic).

dte-deep-reviewer

Review a PR / branch / diff. Multi-lens, cross-model code findings.

dte-test-auditor

Judge a test suite's real value — coverage + quality + mutation, not just presence.

dte-arc-plan

Findings / a spec / a bug → ce-plan-built, ie-validate-plan-validated plans. Decomposes big work into many.

dte-arc-work

Do the work — gated, looped, with a loud report and a pause after every phase.

dte-tooling-scan

Mine history → an overlap-aware shortlist of what tooling to adopt next.

How it works

The discipline a one-off prompt forgets — made first-class.
01

Compose, don't reinvent

Every skill invokes the real tools — ce-plan, ie-validate-plan, layered-rails — not a hand-rolled imitation.

02

Plan, then validate

Real work is planned with compound-engineering and validated with intent-engineering before a line is written.

03

Verify, don't trust

A plan's load-bearing claims are checked against the code, not assumed — the misclassification you catch is the file you'd have moved wrong.

04

Gate + report every step

A runnable gate before every commit; a report and a pause after every phase. No dark loops, no silent failures.

05

Loop at scale

A massive task decomposes into many small validated plans, completed one by one.

06

Degrade gracefully

Optional tools are used if present and their absence is stated — never silently skipped.
